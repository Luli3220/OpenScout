### 项目影响力

引入openrank作为核心指标

**新增讨论热度：** 衡量用户项目下的 `Discussions` 或 `Issues` 被提及的频率，体现项目的持续吸引力。

### 社区贡献（对外）

容易与“代码能力”和“维护者职责”中的**代码产出量**产生重叠。

应该明确聚焦于**对外部项目的贡献**。

**核心指标：** `Accepted Pull Requests Count (to external repos)`（被接受的外部 PR 数量），`Issue Creation Count`（创建的有效 Issue 数量）。

### 代码能力

“规模”与“贡献”重叠，移除规模，专注质量，需要依赖 MaxKB Agent 的微观数据分析。

**核心指标：** 专注于**质量和深度**。

**质量指标：** `Pull Request Review Comments Provided`（提供的代码审查评论数），`Code Acceptance Rate`（代码被合并的成功率，而非 PR 数量），`Revert Rate`（代码被回滚的比例）。

**深度指标（MaxKB 关键）：** 基于微观数据提取的**代码复杂性 (Commit/Patch 颗粒度)**、**架构性修改比例**、以及**高价值技能提取**（由 MaxKB Agent 提取）。

| **维度**                           | **修订后的定义与侧重点**                                     | **核心量化指标（数据来源）**                                 |
| ---------------------------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| 1. 🥇 项目影响力 (Influence) 2      | 衡量用户项目在社区中的**认可度、价值和辐射范围**，侧重于项目的宏观成果。 | **OpenRank 值**（核心指标）、项目 Stars 总数、项目 Forks 总数、项目 Issues/Discussions 的热度。 |
| 2. 🥈 社区贡献 (Contribution) 3     | 衡量用户对**非自己维护的外部项目**的代码和内容产出的勤奋程度。 | **被接受的外部 Pull Request 数量**、创建的有效 Issue 数量、参与外部代码贡献的频率。 |
| 3. 🥉 代码能力 (Code Quality) 4     | 衡量用户**代码产出的质量、深度和专业度**，而不是单纯的数量。通过微观分析体现专业性。 | **MaxKB 提取的高价值技能标签**（微观数据）、代码审查 (PR Review Comments) 数量、代码被合并的成功率、架构性或底层修改的比例。 |
| 4. 🧭 社区互动 (Engagement) 5       | 侧重于用户在社区中的**非代码性、交流性的参与**，体现其社会化技能和帮助性。 | Issue/Discussion 评论数量、通过评论协助他人关闭 Issue 的比例、社区中的问题解决参与度。 |
| 5. 🌍 开源广度 (Diversity) 6        | 衡量用户**知识体系的多元性和参与领域的广泛性**，体现学习和适应能力。 | 核心 Commit 中使用的**不同编程语言数量**、参与的不同项目主题/领域数量、贡献过的不同组织数量。 |
| 6. 🛡️ 维护者职责 (Maintainership) 7 | 衡量用户作为**项目所有者或核心维护者**的项目管理能力和权威性。 | **合并他人 Pull Request 数量**（权威性）、平均 Issue/PR 关闭时间（效率）、对项目文档和健康度文件的贡献。 |
